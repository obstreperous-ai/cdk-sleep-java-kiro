## Test coverage gap â€” `hasResourceProperties` matches one, not all

The test `testBothBucketsHaveSecurityProperties` asserts:

```java
template.resourceCountIs("AWS::S3::Bucket", 2);
template.hasResourceProperties("AWS::S3::Bucket", Map.of(
    "BucketEncryption", ...
    "VersioningConfiguration", ...
    "PublicAccessBlockConfiguration", ...
));
```

`hasResourceProperties` succeeds if *any* resource of that type matches. If the output bucket accidentally lost its encryption config, this test would still pass because the input bucket still matches. The CDK assertions library offers `allResourcesProperties` which asserts that *every* resource of the type matches the pattern. Alternatively, two separate `hasResourceProperties` calls filtered by the notification handler dependency (for input) and its absence (for output) would pin each bucket independently.
